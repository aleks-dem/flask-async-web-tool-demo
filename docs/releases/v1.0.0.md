# v1.0.0 - Public Portfolio Release

Release date: 2026-03-17

## Summary

Initial public release of a sanitized internal-tool demo built with Flask, Celery, and Redis to demonstrate end-to-end delivery and pragmatic engineering decisions.

## Highlights

- Modular Flask architecture with clear domain boundaries.
- Async processing for analyst workflows (lookup + document generation).
- Redis-backed task state with polling UI and progress tracking.
- Dockerized local environment for reproducible setup.
- Test suite covering authentication, access control, route validation, and async service behavior.

## Hardening and Publication Updates

- Removed plaintext demo passwords and switched to hash-only verification.
- Added safer file download path handling to prevent traversal issues.
- Added duplicate-username validation in admin user creation.
- Added Redis TTL for process-state keys.
- Expanded `.gitignore` and removed local runtime artifacts from publication set.

## Documentation and Governance

- Reworked repository `README.md` for portfolio context.
- Added `LICENSE`, `SECURITY.md`, `CONTRIBUTING.md`, and `RELEASE_CHECKLIST.md`.
- Added `CHANGELOG.md` with Keep-a-Changelog structure.
- Added GitHub release workflow for `v*` tags.

## Known Limitations

- Mock data mode is default; no real data source integration is included in this public demo.
- CSRF protection is not enabled yet (acceptable for demo scope, but should be added for internet-facing deployments).
